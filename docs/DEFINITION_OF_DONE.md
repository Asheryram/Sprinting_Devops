# Definition of Done (DoD)

## Overview

The Definition of Done is a shared understanding of what it means for a user story to be complete. A story is only considered "Done" when ALL of the following criteria are met.

---

## Code Criteria

- [ ] **Code Complete** - All acceptance criteria are implemented
- [ ] **Code Reviewed** - Code has been self-reviewed for quality
- [ ] **No Linting Errors** - Code passes ESLint checks
- [ ] **Follows Standards** - Code follows project coding conventions

---

## Testing Criteria

- [ ] **Unit Tests Written** - New functionality has unit tests
- [ ] **Tests Pass** - All unit tests pass locally
- [ ] **Pipeline Passes** - CI pipeline runs successfully

---

## Documentation Criteria

- [ ] **API Documented** - New endpoints are documented in API.md
- [ ] **Code Commented** - Complex logic has explanatory comments
- [ ] **README Updated** - If setup steps change, README is updated

---

## Deployment Criteria

- [ ] **Builds Successfully** - Application builds without errors
- [ ] **No Regressions** - Existing functionality still works
- [ ] **Committed to Main** - Code is merged to main branch

---

## Checklist Summary

Before marking a story as Done:

1. All acceptance criteria met
2. Code reviewed (self-review minimum)
3. Linting passes
4. Unit tests written and passing
5. CI pipeline green
6. Documentation updated
7. Merged to main branch
